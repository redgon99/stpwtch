STPWTCH - 사용자 설명서

1. 소개

STPWTCH는 다기능 타이머 및 스톱워치 애플리케이션입니다. 로컬 환경에서 개인적인 시간 관리는 물론, 서버-클라이언트 모드를 통해 여러 사용자와 타이머 상태를 실시간으로 동기화할 수 있는 고급 기능을 제공합니다. 시험 감독, 그룹 활동, 프레젠테이션 등 다양한 상황에서 활용될 수 있도록 설계되었습니다.

2. 화면 구성

애플리케이션은 크게 세 부분으로 구성됩니다.

  • 왼쪽 패널 (타이머/스톱워치 제어):
    • 모드 및 방 선택 드롭다운 (왼쪽 상단): 모드 선택 (기본), 서버 모드, 클라이언트 모드를 선택하고, 해당 모드에 따라 '방'을 선택하거나 관리합니다.
    • 시간 설정 버튼: 미리 설정된 시간(예: 1분, 5분, 10분 등)으로 타이머를 빠르게 설정합니다.
    • 커스텀 시간 드롭다운: 분 단위로 사용자 정의 시간을 선택하여 타이머를 설정합니다.
    • 제어 버튼:
      • ▶ (시작): 타이머 또는 스톱워치를 시작합니다.
      • || (일시정지): 실행 중인 타이머 또는 스톱워치를 일시정지합니다.
      • ↺ (초기화): 타이머를 설정된 시간으로, 스톱워치를 0으로 초기화합니다.
    • 타이머 표시창: MM:SS.ms (분:초.밀리초) 형식으로 현재 시간을 크게 표시합니다. (주황색 디지털 폰트)
  • 오른쪽 패널 (응시 번호):
    • 응시 번호 라벨: "응시 번호" 텍스트가 표시됩니다.
    • 응시 번호 표시창: 현재 설정된 응시 번호를 큰 숫자로 표시합니다. (0-99)
    • 응시 번호 제어 버튼:
      • +: 응시 번호를 1 증가시킵니다.
      • -: 응시 번호를 1 감소시킵니다.
  • 전체 화면 버튼 (오른쪽 상단): 애플리케이션을 전체 화면으로 전환하거나 원래 크기로 복원합니다.

3. 기본 사용법 (로컬 모드 - 모드 미선택 시)

별도의 모드를 선택하지 않으면 애플리케이션은 로컬 모드로 작동합니다.

3.1. 타이머 사용하기:

  1. 시간 설정:
    • 왼쪽 패널의 시간 버튼 (예: "1분", "5분") 중 하나를 클릭하거나,
    • "커스텀 시간" 드롭다운에서 원하는 시간을 선택합니다.
    • 시간을 선택하면 해당 버튼 또는 드롭다운의 테두리와 글자색이 주황색으로 변경되어 선택되었음을 표시합니다. (타이머는 아직 시작되지 않습니다.)
  2. 시작: ▶ (시작) 버튼을 클릭하면 타이머가 카운트다운을 시작합니다.
  3. 일시정지: || (일시정지) 버튼을 클릭하면 타이머가 잠시 멈춥니다. 다시 ▶ 버튼을 누르면 이어서 진행됩니다.
  4. 초기화: ↺ (초기화) 버튼을 클릭하면 타이머가 마지막으로 설정된 시간으로 돌아가고 정지 상태가 됩니다. 선택되었던 시간 버튼/드롭다운의 주황색 표시도 사라집니다.
  5. 타이머 종료 시: 시간이 다 되면 타이머 표시창이 "00:00.00"으로 변경되고, 선택되었던 시간 버튼/드롭다운의 주황색 표시가 사라집니다.

3.2. 스톱워치 사용하기:

  1. 모드 진입: 특정 시간을 설정하지 않은 상태 (예: 페이지 초기 로드 상태 또는 타이머를 초기화하여 timerDuration이 0인 상태)에서 ▶ (시작) 버튼을 클릭하면 스톱워치 모드로 작동하여 시간이 카운트업됩니다.
  2. 일시정지/초기화: 타이머와 동일하게 || (일시정지), ↺ (초기화) 버튼으로 제어합니다. 초기화 시 스톱워치는 "00:00.00"으로 돌아갑니다.

3.3. 응시 번호 변경:

오른쪽 패널의 + 또는 - 버튼을 클릭하여 응시 번호를 0부터 99까지 변경할 수 있습니다.

3.4. 전체 화면 모드:

화면 오른쪽 상단의 전체 화면 아이콘을 클릭하여 전체 화면으로 보거나 원래대로 돌아올 수 있습니다.

4. 고급 기능 (서버/클라이언트 모드)

이 기능을 사용하면 타이머, 스톱워치, 응시 번호 상태를 다른 사용자와 실시간으로 공유할 수 있습니다. (Supabase 백엔드 서비스 연동 필요)

4.1. 모드 선택:

화면 왼쪽 상단의 첫 번째 드롭다운 메뉴에서 서버 모드 또는 클라이언트 모드를 선택합니다.

4.2. 서버 모드:

  1. 활성화:
    • 모드 드롭다운에서 서버 모드를 선택합니다.
    • 두 번째 드롭다운 (방 선택)이 나타나며, 기존 방 목록이 로드됩니다. (방 상태: 🟢 활성, ⚫ 비활성)
    • 새로운 세션을 시작하려면 방 목록에서 기존 방을 선택하거나, 시스템에서 자동으로 새 방 번호가 할당될 수 있습니다. (구현에 따라 다름)
    • 방이 성공적으로 선택/활성화되면 "서버 모드 (방번호: XXX)"와 같이 표시되며, 해당 방은 활성(🟢) 상태가 됩니다.
  2. 사용법: 로컬 모드와 동일하게 타이머/스톱워치 및 응시 번호를 제어합니다. 모든 변경 사항 (시간 값, 실행 상태, 모드, 응시 번호)이 서버에 주기적으로 저장되어 클라이언트와 동기화됩니다.
  3. 비활성화: 모드 드롭다운에서 다른 모드를 선택하거나 "모드 선택"으로 돌아가면 서버 모드가 비활성화됩니다. 이때 해당 방은 비활성(⚫) 상태로 변경되고, 로컬 타이머 및 응시 번호 등 모든 상태가 초기화됩니다.

4.3. 클라이언트 모드:

  1. 연결:
    • 모드 드롭다운에서 클라이언트 모드를 선택합니다.
    • 방 선택 드롭다운에 현재 활성화된(🟢) 서버 세션(방) 목록이 표시됩니다. 비활성화된(⚫) 방은 선택할 수 없습니다.
    • 연결하고자 하는 활성 방을 선택합니다.
    • 성공적으로 연결되면 "클라이언트 모드 (방번호: XXX)"와 같이 표시됩니다.
  2. 사용법:
    • 클라이언트 모드에서는 타이머, 스톱워치, 응시 번호 관련 모든 제어 버튼(시간 설정 버튼, 시작/일시정지/초기화, 응시 번호 +/-)이 비활성화됩니다.
    • 화면 표시는 연결된 서버의 상태를 실시간으로 반영합니다. 서버에서 타이머가 시작되면 클라이언트 화면에도 동일하게 적용됩니다.

5. 표시 정보

  • 시간 표시: MM:SS.ms (예: 01:23.45는 1분 23초 450밀리초)
  • 응시 번호: 0부터 99까지의 숫자 (예: 07)
  • 방 상태 표시 (서버/클라이언트 모드 시):
    • 🟢: 해당 방이 현재 활성 상태임을 의미합니다 (서버가 실행 중이거나, 클라이언트가 연결 가능한 상태).
    • ⚫: 해당 방이 비활성 상태임을 의미합니다.
